# 06 - Validation and Change

## Tujuan
Dokumen ini digunakan untuk memvalidasi requirement dan mencatat perubahan requirement selama proyek berjalan.

## Requirement Validation Checklist

| ID | Checklist | Status | Catatan |
|---|---|---|---|
| VC-01 | Setiap functional requirement memiliki ID. | Valid | FR-01 sampai FR-16 sudah dibuat. |
| VC-02 | Jumlah functional requirement minimal 12. | Valid | Terdapat 16 functional requirements. |
| VC-03 | Jumlah non-functional requirement minimal 6. | Valid | Terdapat 8 non-functional requirements. |
| VC-04 | Jumlah business rule minimal 5. | Valid | Terdapat 8 business rules. |
| VC-05 | Jumlah user story minimal 10. | Valid | Terdapat 14 user stories. |
| VC-06 | Setiap user story memiliki acceptance criteria. | Valid | Setiap user story memiliki minimal 2 acceptance criteria. |
| VC-07 | Fitur sesuai scope tugas dosen. | Valid | Fitur non-wajib ditunda. |
| VC-08 | Requirement dapat ditelusuri ke issue dan test. | Perlu lanjut | Akan dibuat pada traceability matrix. |

## Validasi Scope
Fitur yang masuk scope:
- Membuat laporan.
- Melihat daftar laporan.
- Mencari dan filter laporan.
- Melihat detail laporan.
- Admin review laporan.
- Admin menentukan prioritas.
- Admin menugaskan teknisi.
- Teknisi update status.
- Komentar/catatan.
- Riwayat status.
- Menutup dan membuka kembali laporan.
